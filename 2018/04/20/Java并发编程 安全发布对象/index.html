Java并发编程 安全发布对象

发布对象

  • 使一个对象能够被当前范围之外的代码所使用

对象逸出

  • 一种错误的发布.当一个对象还没构造完成时,就被其它线程所见

如何安全发布对象?

  • 在静态初始化函数中初始化一个对象
  • 将对象引用保存到volatile类型域或者AtomicReference对象中
  • 将对象引用保存到某个正确构造对象的final类型域中
  • 将对象引用保存到一个由锁保护的域中

几种单例模式

饿汉模式(单例实例在装载时进行创建)


懒汉模式(单例实例在第一次使用时创建)

  • 这种方法在多线程环境下存在 Bug , JVM 和 CPU 优化,发生了指令重排列
  • 使用 volatile , 禁止指令重排列实现线程安全
